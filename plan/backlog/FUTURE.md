# Future Enhancements

This document tracks features planned for future iterations. These are not currently prioritized but represent valuable improvements for SandVoice.

---

## Alternative LLM/TTS/STT Providers

The provider facade (Plans 41–46) is in place. Adding new provider implementations keeps callers unchanged, but `common/ai.py` still needs to be extended to register/select the new provider (updating `_SUPPORTED_PROVIDERS` and the `_build_*_provider` dispatch dicts), and config validation may also need updates. Candidates:

- **LLM**: Anthropic Claude, Google Gemini, local Ollama (for Pi deployments)
- **TTS**: ElevenLabs (higher quality voices), Piper / Coqui (offline, Pi-friendly)
- **STT**: faster-whisper or whisper.cpp (offline, lower latency on Pi)

Each new provider is a class implementing the relevant ABC plus the required factory wiring in `common/ai.py`.

---

## Local Offline Mode (Raspberry Pi)

Combine local STT (faster-whisper), local LLM (Ollama + small model), and local TTS (Piper) into a fully offline configuration. No API keys, no latency, works without internet. Particularly valuable for Pi deployments. Builds directly on the Alternative Providers idea above.

---

## Conversation History Management

Conversation history grows unbounded across a session, which will eventually hit LLM token limits. Implement smart truncation: keep the last N turns verbatim, summarize older turns into a single context message. The summary itself can be generated by `ai.one_shot()`.

---

## Timers & Reminders (User-Triggered)

The Task Scheduler (Plan 21) handles cron/interval/once scheduled tasks defined in config. What's missing is *voice-triggered* timers: "remind me in 10 minutes to check the oven." Requires parsing relative time from user input and dynamically registering a one-shot scheduler task at runtime.

---

## Conversation Memory (Cross-Session)

Persistent memory of facts across sessions: "My favorite coffee shop is Starbucks on Main Street" → later: "directions to my favorite coffee shop." Likely implemented as a dedicated SQLite table with semantic search or simple keyword lookup via `ai.one_shot()`. Privacy controls for what gets stored.

---

## Conversation Export

Save the current session's conversation history to a file (Markdown or plain text) on request or on exit. Low effort, useful for reviewing interactions or debugging misroutes.

---

## Plugin Hot-Reload

Reload the plugin registry (and optionally `routes.yaml`) without restarting the process. Useful during development and for config-only plugin changes. Could be triggered via a voice command or a signal.

---

## API Cost Tracking

Track OpenAI API usage (Whisper STT, LLM completions, TTS) per session and cumulatively. Log token counts and estimated cost. Optionally warn when approaching a configurable daily/monthly budget threshold.

---

## Music Control

Integration with music services or local playback:
- Spotify API for "play my workout playlist"
- Local music library (MPD)
- Basic controls: play, pause, skip, volume
- Natural language queries: "play something upbeat"

---

## Smart Home Integration

Control smart home devices through voice:
- HomeAssistant or HomeBridge integration
- "Turn off living room lights", "set thermostat to 20 degrees", "is the garage door closed?"

---

## Calendar Integration

Access calendar via natural language:
- "What's on my calendar today?", "When is my next meeting?"
- Google Calendar, iCloud, Outlook API support

---

## Todo / Shopping List

Manage lists by voice:
- "Add milk to the shopping list", "What's on my todo list?", "Mark call dentist as done"
- Local storage or Todoist / Things integration

---

## Multi-User Support

Recognize different users by voice fingerprint and provide personalized responses (calendar, preferences, todo lists). Privacy-first design for shared devices.

---

## Notes
